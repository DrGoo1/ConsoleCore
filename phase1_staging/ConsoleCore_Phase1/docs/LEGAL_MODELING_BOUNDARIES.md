# Legal-Safe Console Modeling Boundaries

ConsoleCore should model console behavior without copying proprietary source code, protected branding, trade dress, schematics under restrictive license, or distinctive commercial presets.

## Safe Approach

Use public-domain engineering principles and original measurement data:

- transformer saturation behavior
- tube/nonlinear stage approximations
- op-amp slew behavior
- bus rail sag/recovery
- VCA-style dynamics behavior
- channel tolerance variation
- crosstalk/coupling matrices
- frequency-dependent phase shift
- noise floor and headroom behavior

## Naming Rule

Use descriptive family names unless licensed:

- British Class-A Discrete
- British VCA Large Format
- American Discrete Op-Amp
- German Tube Broadcast
- German Solid-State Broadcast
- EMI-Inspired Tube Desk
- Flickinger-Inspired Discrete Console
- Modern Ultra-Clean

Avoid product claims such as “exact SSL 4000,” “exact Neve 8068,” or “exact API Legacy” unless legally reviewed and backed by original measurement data.

## Reference Use

Third-party plugins, JSFX, and hardware documentation may be used for:

- listening comparisons
- behavioral benchmarking
- architectural inspiration
- validating that ConsoleCore is musically useful

Do not copy:

- source code
- parameter curves
- exact GUI layouts
- proprietary names as product model names
- protected artwork or manuals
